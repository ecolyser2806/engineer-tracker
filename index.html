⚙ Setup Google Sheets
One-time configuration — takes about 5 minutes

Connect to Google Sheets

Follow these steps once. After setup, all data saves automatically to your Google Sheet and can be viewed or edited by anyone you share it with — from any device.

1
Open Google Sheets and create a new blank spreadsheet. Name it Ecolyser Incentive Tracker.
2
In that sheet, click Extensions → Apps Script. A new tab opens with a code editor.
3
Delete everything in the editor. Open the google_apps_script_v4.js file you downloaded and paste its entire contents. Click Save (Ctrl+S). Name the project PTTrack v4.
4
Click Deploy → New Deployment. Set Type = Web App. Set Execute as = Me. Set Who has access = Anyone. Click Deploy. Google will ask you to authorise — click Allow.
5
Copy the Web App URL that appears (starts with https://script.google.com/macros/s/...). Paste it below and click Connect.
🔌 Connection Not Working? — Most Common Fixes
① Did you create a NEW deployment? — You must use Deploy → New Deployment each time you update the script. Editing an existing deployment does NOT update the live URL.
② "Who has access" must be set to "Anyone" — Not "Anyone with a Google Account". Open Apps Script → Deploy → Manage Deployments → Edit → fix this.
③ Did you click "Authorise"? — On first deploy, Google shows a permissions screen. You must click Allow. If you missed it, delete the deployment and redeploy.
④ Using the old URL? — Every new deployment gets a NEW URL. Paste the latest URL from Deploy → Manage Deployments here.
⑤ Still failing? — Open your Apps Script URL directly in a browser tab. If it shows JSON data, it's working. If it shows an error, the issue is in the script permissions.
📝 After Connecting — How to Edit Data & Share Access
• Open your Google Sheet anytime from any device or browser
• You will see 4 tabs: Engineers, Monthly Records, Daily Logs, and Sites
• Click any cell and edit directly — changes are saved instantly
• Click 🔄 Refresh in this tracker to reload the latest data

🔗 Sharing this tracker with your team:
• This HTML file works as a standalone web app — host it on any web server, Google Drive (Publish to web), or share the file directly
• Anyone with the file + the Google Apps Script URL can use it fully from any laptop or mobile
• The Google Apps Script URL is set to "Who has access: Anyone" — this means anyone with the URL can read/write data to your sheet
• To restrict access, change the deployment to "Anyone with Google Account" and team members sign in

⚠ Important: The Google Apps Script Web App URL is the "key" to your data. Keep it within your team.

🛡 How to Verify Checklist Integrity Without Manual Checking
★★★ HOD-Critical Items
Every item marked HOD-critical (red badge) has a named evidence trail — email timestamps, signed service reports, shared Drive photos, monitoring screenshots. Cross-check these directly in your email / shared folder instead of trusting the tick.
⚠ Anomaly Flags
The system auto-flags: logs saved >48 hrs after visit date, perfect 100% scores on emergency visits, high score with signature/photo unchecked, emergency visits with no remarks. Each flag tells you exactly what to verify.
📅 Visit Completion Rate
Compare scheduled visits logged in daily log vs sites managed declared in monthly record. Low completion rate directly reduces quarterly incentive — engineers cannot inflate the sites count without actual visit logs matching.
📊 Score Trend
A sudden jump from 30% to 95% between months without a training event is itself a flag. Consistent scores across visits are more credible than one-off perfect submissions. Review the per-visit breakdown below.
